Install Ord

Note: Cargo presents a simple method for installing Ord. While we did not try this, instead of using Rust/Cargo you may be able to install the latest pre-built binary from Terminal with the following command.

curl --proto '=https' --tlsv1.2 -fsLS https://ordinals.com/install.sh | bash -s

The method chosen below uses Cargo to install Ord.

Install Cargo with RustUp

Ord is written in Rust. RustUp installs the required Rust language and Cargo package manager. Commands below are to be used in Terminal.

  1. Go to https://rustup.rs and copy the command and paste & run it in Terminal
  2. Where you are presented an option and one is shown with “(default)”, just press enter at the prompt to use that default
  3. Add Cargo to the path
    1. First if the following command returns "cargo not found", that confirms Cargo is not already in the shell.
which cargo

source "$HOME/.cargo/env"
  1. Try again to confirm it now finds Cargo
which cargo
  1. Close Terminal and reopen it

Create Aliases for Use in Terminal

With your Bitcoin Core cookie file and your Ord index files in non-standard locations (external HD), you need to use the --cookie-file and --index arguments with every Ord command you run (e.g. updating the index or starting the server). To make this easier, you can create alias commands for use in Terminal. Note: The below uses zsh. Similar steps exist to do the same with bash.)

• Open your zshrc file in Terminal: nano ~/.zshrc • In the .zshrc file, create an alias by adding a line with the following format: • alias alias_name='command_to_execute' o Replace alias_name with the name you want to give to your alias, and command_to_execute with the command you want to execute when you use your alias.  alias ordindex='/Users/mattlhx/.cargo/bin/ord --index /Volumes/Bitcoin/Ord/index.redb --cookie-file /Volumes/Bitcoin/Bitcoin/.cookie index'  alias ordserver='/Users/mattlhx/.cargo/bin/ord --index /Volumes/Bitcoin/Ord/index.redb --cookie-file /Volumes/Bitcoin/Bitcoin/.cookie server'  alias bitcoindm='/opt/homebrew/Cellar/bitcoin/24.0.1/bin/bitcoind -datadir=/Bitcoin/Bitcoin'  Save the changes to the .zshrc file by pressing Ctrl+O and then Enter o Exit the nano editor by pressing Ctrl+X • To make the changes take effect, you can either restart your Terminal application, or type source ~/.zshrc to reload the .zshrc file and make your new alias available for use o Now, whenever you type your alias in the Terminal, it will execute the command you specified in the .zshrc file.

Create an Ord Wallet & Inscribe

Reference: Hell Money Podcast: How to Make an Inscription Part 2

• Create the wallet o Kill the server with CTRL-C o To create the wallet: ord --index /Volumes/Bitcoin/Ord/index.redb --cookie-file /Volumes/Bitcoin/Bitcoin/.cookie wallet create | pbcopy (Note: Add ‘ | pbcopy’ to the end of the command if you want the resulting seed phrase to automatically be copied to the clipboard o Temporarily paste the seed phrase as desired (Careful… you know the deal with seed phrases) • Create an inscription o To get a fresh address: ord --index /Volumes/Bitcoin/Ord/index.redb --cookie-file /Volumes/Bitcoin/Bitcoin/.cookie wallet receive o Send BTC to this address (to cover inscription fees)  Ord will play a sound when the transaction is confirmed  Wait for QT to show the updated available balance  Or go to mempool.space and paste the address to see the status of the transaction  To confirm with Ord: ord --index /Volumes/Bitcoin/Ord/index.redb --cookie-file /Volumes/Bitcoin/Bitcoin/.cookie wallet balance o ord --index /Volumes/Bitcoin/Ord/index.redb --cookie-file /Volumes/Bitcoin/Bitcoin/.cookie wallet inscribe /Users/mattlhx/Desktop/OrdinalsTest.txt --fee-rate 20  This test is to create a text-based Ordinal with the contents of filename.txt as the body of the Ordinal  Wait for the commit and reveal transactions to complete • Note your inscription ID is shown in the output as well. You can paste that into Ordinals.com or your Ord page to see the inscription  The inscription is sent to a new address in the wallet  To do a dry run (perhaps to see the total cost before actually broadcasting the transaction): ord --index /Volumes/Bitcoin/Ord/index.redb --cookie-file /Volumes/Bitcoin/Bitcoin/.cookie wallet inscribe /Users/mattlhx/Desktop/OrdinalsTest.txt --destination bc1pqlgmfc3v84ksw0e2603tcswqutllzxm99xl4luavvsdxdudtcn0svkqe78 --fee-rate 20 --dry-run • Another example: ord --index /Volumes/Bitcoin/Ord/index.redb --cookie-file /Volumes/Bitcoin/Bitcoin/.cookie wallet inscribe /Users/mattlhx/Desktop/Fool\ 151\ Pop\ Artist.webp --destination bc1pkuqhjz9298gwwfr7yewg47a3gr5hgqmu247h2vh6m9jkp269w68svlwwx7 --fee-rate 15 --dry-run o To see your inscription: ord --index /Volumes/Bitcoin/Ord/index.redb --cookie-file /Volumes/Bitcoin/Bitcoin/.cookie wallet inscriptions • Transfer the new Ordinal to a keeper wallet o For the safety afforded with its coin control features, this is most easily done from Sparrow where you have imported this Ord wallet using the seed phrase
